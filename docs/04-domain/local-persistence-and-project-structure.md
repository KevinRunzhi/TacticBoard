# Local Persistence And Project Structure

最后更新：2026-04-01

## 1. 持久化原则

- 第一阶段所有核心数据保存在本地
- 不依赖云端、账号或远程同步
- 应用启动和打开项目不依赖网络
- Project 持久化内容包含 MatchMeta 与项目级 ReferenceImage

## 2. 本地对象边界

正式持久化：

- Project
- FormationTemplate
- ProjectIndexEntry

恢复性持久化：

- LocalDraft

仅当前会话的临时状态：

- ExportConfig（仅当前导出会话使用，不进入本地持久化，不跨会话保留）

## 3. 素材处理规则

- 球员头像导入后必须复制到应用本地存储
- 参考底图导入后必须复制到应用本地存储
- 不允许长期依赖外部路径或临时授权引用

## 4. 失败处理规则

- 自动保存失败时保留上一份成功草稿
- 素材导入失败时不得留下空引用
- 项目记录失效时允许移除无效条目
- 最近项目索引失效时允许单独清理，不阻塞进入工作台

## 5. 启动恢复顺序

建议顺序：

1. 读取项目索引与最近项目索引
2. 清理明显无效的索引项
3. 读取可恢复草稿信息
4. 再决定是否弹出恢复入口

规则：

- 启动顺序不能因为某个草稿损坏而阻塞首页
- 索引层异常不应直接判定正式项目损坏

## 6. 第一阶段明确不做

- 项目文件导入导出
- 本地备份包
- 历史版本
- 跨设备迁移保证

## 7. 后续扩展约束

- 若后续引入项目文件，必须内嵌素材
- 若后续引入外部导入，同名导入默认保留原项目并生成可区分副本名
