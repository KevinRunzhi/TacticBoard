# Persistence Strategy

最后更新：2026-03-31

## 1. 总原则

- V1 采用本地优先持久化
- 先保证可靠写入，再考虑复杂迁移

## 2. 写入对象

- Project
- FormationTemplate
- LocalDraft
- ProjectIndexEntry

说明：

- `MatchMeta` 和项目级 `ReferenceImage` 作为 `Project` 的组成部分随 `Project` 一起写入，不单独持久化

## 3. 写入时机

Project：

- 手动保存时写入
- 项目复制时生成新项目写入
- 项目重命名时更新元数据
- 写入内容包含 `matchMeta`、`referenceImage` 与当前全部步骤内容

LocalDraft：

- 编辑器关键变更后写入
- 用于恢复最近进度

ProjectIndexEntry：

- 项目创建、重命名、复制、打开、删除时更新

## 4. 自动保存策略

- 编辑器在关键变更后触发自动保存
- 自动保存仅覆盖当前工作副本
- 失败时保留上一份成功草稿
- 自动保存失败不应阻塞继续编辑
- 自动保存不应覆盖用户已手动保存的项目状态
- 第一阶段建议把“会进入撤销栈的内容变更”作为自动保存触发条件
- 第一阶段建议采用去抖写入：最后一次内容变更后约 2 秒触发一次自动保存
- 连续拖拽或持续编辑时，建议设置最长约 10 秒的强制落盘上限，避免长时间完全不写入
- 步骤切换、应用进入后台或用户触发手动保存前，应先冲刷待写入的自动保存队列

## 5. 恢复策略

- 启动后发现存在有效草稿时，提供恢复或跳过入口
- 跳过恢复不应删除正式项目
- 草稿损坏时允许忽略并清理异常记录
- 恢复判断应在索引清理之后进行
- 最近项目索引异常不应阻塞进入首页

## 6. 素材持久化策略

- 导入头像与底图后复制到应用本地存储
- 不依赖外部文件路径长期存在
- 素材导入失败不保留空引用

## 7. 删除策略

- 删除项目前二次确认
- 删除项目后从最近项目中同步移除
- 删除项目后相关草稿应同步清理

## 8. 不进入持久化的状态

- 当前选中对象
- 当前工具
- 播放状态
- 抽屉开关
- 画布临时缩放与视角
- 导出弹窗临时输入

## 9. 当前不做

- 跨设备迁移保证
- 项目文件格式
- 本地备份包
- 历史版本仓库
