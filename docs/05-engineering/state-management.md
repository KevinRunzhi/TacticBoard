# State Management

最后更新：2026-03-31

## 1. 状态分类

### 全局应用状态

- 当前路由
- 当前页面壳层状态
- 通知与提示

### 项目列表状态

- 最近项目列表
- 项目排序
- 项目基础操作

### 编辑器状态

- 当前项目
- 当前步骤索引
- 当前工具
- 当前选中对象
- 播放状态
- 撤销重做栈
- 画布缩放与视角

### 短期 UI 状态

- 抽屉开关
- 浮层开关
- 临时表单输入

## 2. 状态归属规则

- 项目数据归编辑器状态拥有
- 页面导航状态归壳层拥有
- 短期 UI 状态不要进入持久化
- 导出参数默认作为临时状态管理

## 3. 状态归属矩阵

| 状态类型 | 归属位置 | 是否持久化 |
|---|---|---|
| 最近项目列表 | 项目列表状态 | 是 |
| 当前项目内容 | 编辑器状态 | 是 |
| 当前步骤索引 | 编辑器状态 | 是 |
| 撤销重做栈 | 编辑器状态 | 否 |
| 当前选中对象 | 编辑器状态 | 否 |
| 当前工具 | 编辑器状态 | 否 |
| 播放状态 | 编辑器状态 | 否 |
| 画布缩放与视角 | 编辑器状态 | 否 |
| 抽屉开关 | 短期 UI 状态 | 否 |
| 导出参数 | 短期 UI 状态 | 否 |
| 自动保存草稿 | 持久化层 | 是 |

## 4. 状态更新原则

- 项目内容变化后，编辑器状态是唯一真相来源
- 持久化层只负责保存和恢复，不重新定义项目真相
- 页面切换不应偷偷改写项目内容
- 导出逻辑读取项目状态，但不拥有项目状态
- 播放状态、选中状态、缩放视角等派生展示状态默认不进入项目持久化

## 5. 撤销 / 重做边界

- 撤销栈属于编辑器状态
- 页面切换和抽屉开关不进入撤销栈
- 对象移动、文本修改、步骤编辑进入撤销栈

## 6. 反模式

- 多个页面同时维护项目真相
- 把 LocalDraft 当成正式项目
- 把页面开关状态写入项目数据
- 在导出流程中直接修改项目显示状态
- 把播放状态或缩放视角当作需要永久保存的项目字段
