# 足球战术板需求记录

最后更新：2026-03-31

## 文档用途

这份文档用于记录足球战术板第一阶段的当前需求基线。
它服务于当前要做的产品目标：`Windows + Android` 首发、`离线单机`、`免注册`、`业余足球爱好者优先`。

用途包括：
- 作为第一阶段实现范围的细化说明
- 作为 PRD 的落地补充
- 作为后续讨论“哪些要做、哪些不做、哪些往后放”的统一依据

说明：
- 本文档聚焦第一阶段可交付范围
- 后续增强能力会单独列出，不默认纳入首发
- 如与 PRD 冲突，应先更新 PRD，再同步这份文档

## 当前产品定义

一个面向业余足球爱好者、业余球队组织者和普通球迷的跨端本地战术板应用，支持在 Windows 与 Android 设备安装运行，用来快速制作阵型图、战术讲解图和回合复盘图，并通过本地保存、图片导出和基础 GIF 导出完成日常使用。

当前阶段说明：
- 内容创作者场景作为兼容方向保留，但不是第一阶段主目标
- 第一阶段不做在线账号、在线分享页、云同步和实时协作
- 第一阶段优先完成“新建 -> 编辑 -> 保存 -> 导出”的核心创作闭环

## 当前产品原则

- 第一阶段先把核心创作闭环做完整，再逐步扩展专业能力
- Windows 与 Android 首发，但不要求两个平台在所有专业效率能力上完全对齐
- 核心编辑、保存和导出默认离线可用，不依赖网络远程拉取
- 第一阶段不引入远程模板、远程主题、远程素材拉取等联网依赖
- 第一阶段优先保证稳定、清晰、易用，而不是追求大量专业增强能力
- 保存与编辑不应误覆盖用户已有内容
- 范围控制优先于功能贪多，所有超出主链路的能力都应先进入后续评估

## 当前已确认

### 1. 产品方向

- 产品类型：足球战术板
- 产品形态：跨端本地应用
- 首发平台：Windows、Android
- 首发人群：业余足球爱好者、业余球队组织者、普通球迷
- 当前不做：注册、登录、云同步、分享链接、实时协作

### 2. 核心诉求

- 比现有工具更适合表达阵型、跑位和战术意图
- 比现有工具更适合快速导出清晰的战术图
- Windows 端支持基础 GIF 导出，用于回合演示
- 应用安装后即可本地使用，不需要注册和联网
- 用户能够继续编辑最近项目，不必每次从零开始

### 3. 主要使用场景

- 队内讨论阵型和战术
- 做简单讲解图和队内分享图
- 赛后复盘某个回合，并通过步骤播放、图片导出或 GIF 导出展示过程

说明：
- 第一阶段不以“社媒内容包装”作为主要场景
- 第一阶段不以“长期资产管理平台”作为主要场景

### 4. 平台与分发

第一阶段确认：

- GitHub Releases 提供 Windows 安装包
- GitHub Releases 提供 Android APK
- 首发阶段采用手动下载安装和手动更新
- 目标支持 Windows 10 及以上
- 目标支持 Android 10 及以上

第一阶段不做：

- 应用商店首发分发
- 应用内自动更新
- 应用内静默更新

规则：

- Windows 安装版优先，便携版不是第一阶段必做项
- Android 首发以 Releases 下载 APK 为主
- 应用升级默认不得清空本地项目和自定义阵型模板
- 如安装方式存在数据风险，必须在更新说明中明确提示

### 5. 离线与本地保存

第一阶段确认：

- 首次打开不强制注册或联网
- 核心编辑、保存、PNG 图片导出在无网络条件下必须可正常使用
- Windows 端 GIF 导出在无网络条件下必须可正常使用
- 本地自动保存草稿
- 手动保存当前项目
- 最近项目列表
- 最近项目继续编辑
- 自定义阵型模板保存在本地

规则：

- 本地自动保存优先覆盖当前工作副本，不污染当前项目的可恢复状态
- 应用异常退出后，下次启动优先提示恢复最近一次自动保存内容
- 自动保存写入失败时，必须保留上一份成功写入的本地草稿
- 应用启动、进入编辑器和打开最近项目不得被联网状态检查、远程拉取或更新检查阻塞
- 删除本地项目之前必须二次确认
- 项目复制作为第一阶段替代版本管理的轻量方案
- 第一阶段不保证跨设备迁移；卸载应用、清理应用数据或更换设备可能导致本地项目丢失，需在应用内明确提示用户

第一阶段不纳入：

- 项目文件导入导出
- 本地备份包
- 历史版本与本地快照
- 完整本地资产库

后续约束：

- 若后续纳入项目文件或本地备份包，必须内嵌球员头像、参考底图等本地素材，不允许只记录设备本地路径
- 若后续纳入项目文件导入导出，旧版本文件可迁移打开，高于当前版本的文件需明确提示兼容风险
- 若后续纳入项目文件导入导出，同名导入时默认保留原项目，并为新导入副本生成可区分名称

### 6. 场地模板

第一阶段确认支持：

- 11 人制全场 / 半场
- 8 人制全场 / 半场
- 7 人制全场 / 半场
- 5 人制全场 / 半场

轻量场地自定义支持：

- 横版 / 竖版切换
- 显示 / 隐藏部分线条
- 首版提供 2 套球场视觉风格：简洁平面 / 草坪写实

第一阶段不做：

- 完全自由绘制球场

### 7. 阵型模板

第一阶段确认：

- 阵型支持预设模板
- 用户可以拖动预设阵型里的每个球员位置
- 用户可以把调整后的站位保存成自己的阵型模板
- 阵型模板只包含站位结构，不包含文本、线路、步骤

11 人制首版内置阵型：

- 4-3-3
- 4-2-3-1
- 4-4-2
- 4-4-2 菱形中场
- 4-5-1
- 4-1-4-1
- 4-3-1-2
- 3-5-2
- 3-4-3
- 3-4-2-1
- 3-4-1-2
- 5-3-2
- 5-4-1

小场制首版内置阵型：

- 8 人制：3-3-1、3-2-2、2-3-2、4-2-1
- 7 人制：2-3-1、3-2-1、2-1-2-1
- 5 人制：2-2、1-2-1、2-1-1、1-1-2

制式切换规则：

- 切换到不兼容制式时必须先提示
- 只提供：取消切换 / 保留当前对象并切换球场 / 按新制式重新生成阵型
- 第一阶段不做自动删除多余球员
- 第一阶段不做智能重排
- 第一阶段不做自动映射到新制式

### 8. 核心编辑对象

第一阶段支持：

- 球员
- 足球
- 文本说明
- 区域标记
- 参考底图

#### 8.1 球员对象

第一阶段确认：

- 颜色
- 号码
- 名字或简称
- 头像
- 用户上传真实头像
- 上传后支持裁切和缩放
- 球衣卡片式支持显示已上传头像
- 位置标签
- 位置标签支持英文缩写和中文
- 队伍归属
- 首版展示样式仅支持简洁圆点式、球衣卡片式

规则：

- Android 导入球员头像时必须优先使用系统文件选择器，并通过系统授权流程明确申请权限
- Android 端首发不默认申请广泛存储权限，优先使用系统文件选择器完成最小化授权
- 首阶段导入的球员头像必须复制到应用本地存储，不得长期依赖外部文件路径或临时授权引用
- 素材导入失败时，不得留下不可用的空引用

第一阶段不做：

- 独立中立球员 / 占位对象
- 训练器材对象

#### 8.2 足球对象

第一阶段确认：

- 足球对象支持拖拽移动
- 足球对象支持经典足球样式 / 简洁圆点样式
- 足球对象支持大小调整
- 足球对象支持与球员对象吸附
- 第一版只支持单个足球对象

第一阶段不做：

- 足球对象名字
- 足球对象号码
- 足球对象头像

#### 8.3 文本说明对象

第一阶段确认：

- 可用于标题、注释、简短说明、战术解释
- 文本仅支持 标题 / 正文说明 / 强调 三种预设样式
- 支持字号、颜色、粗细、对齐
- 支持无背景 / 纯色底
- 支持基础边框和圆角
- 支持透明度

第一阶段不做：

- 富文本
- 段落级复杂排版
- 旋转文字

#### 8.4 区域标记对象

第一阶段确认：

- 支持矩形区域
- 支持圆形 / 椭圆区域
- 支持边框、填充色、透明度

第一阶段不做：

- 自由多边形区域

#### 8.5 参考底图

第一阶段确认：

- 上传图片作为参考底图
- 参考底图作为项目级对象，默认跨全部步骤共享
- 参考底图缩放、拖动、裁切
- 参考底图透明度调节
- 参考底图锁定、隐藏、显示切换

规则：

- Android 导入参考底图时必须优先使用系统文件选择器，并通过系统授权流程明确申请权限
- Android 端首发不默认申请广泛存储权限，优先使用系统文件选择器完成最小化授权
- 首阶段导入的参考底图必须复制到应用本地存储，不得长期依赖外部文件路径或临时授权引用
- 素材导入失败时，不得留下不可用的空引用

第一阶段不做：

- 直接视频导入并截帧

### 9. 线路系统

第一阶段支持：

- 跑位箭头
- 传球线路
- 带球线路
- 射门线路
- 防守压迫方向

要求：

- 第一版支持直线
- 第一版支持折线
- 第一版不支持曲线 / 贝塞尔曲线
- 第一版不支持弧线
- 起点和终点支持自动吸附到球员 / 足球对象
- 不同线路类型默认具备不同视觉样式
- 在线条颜色、粗细、虚实层面可直接区分语义

### 10. 步骤演示

第一阶段支持：

- 多步骤帧
- 复制上一帧继续编辑
- 步骤播放
- 上一帧残影显示
- 步骤标题
- 步骤说明 / 讲解备注
- 步骤时间标签 / 回合标签
- 步骤缩略导航
- 导出时可选择带上步骤信息

规则：

- 复制上一帧时，球员、足球、文本、区域、线路对象继承同一对象身份
- 相邻步骤帧通过对象身份建立对应关系，用于残影和步骤连续性表达
- 在某一帧删除对象，不影响更早的历史帧
- 如果后续帧从当前帧复制而来，则被删除对象默认继续不存在
- 用户手动重新新增的对象视为新对象身份

### 11. 比赛信息与项目元数据

第一阶段支持：

- 基础比赛信息
- 比分、分钟、回合阶段
- 比赛信息展示在标题区、项目预览和导出图中

第一阶段不做：

- 比赛信息参与搜索和筛选
- 训练场景专用元数据

### 12. 图片与 GIF 导出

第一阶段支持：

- PNG 导出
- 多种导出比例
- 高清导出
- Android 端通过系统分享发送导出图片
- GIF 导出，Windows 首发优先

输出增强：

- 导出无背景透明图层
- 导出仅球员和线路的内容组合
- 标题区
- 图片导出比例：16:9 / 1:1 / 9:16
- 标题 / 步骤说明 / 比赛信息统一按开关选择是否带上

GIF 首发约束：

- GIF 仅基于步骤播放结果导出
- 不做镜头动画
- 不做音频
- 不做逐帧精修
- 只提供 慢 / 标准 / 快 三档速度
- 首发总时长上限为 15 秒
- Windows 端将 GIF 导出纳入第一阶段
- Android 端首版不强制支持 GIF 导出，优先保证 PNG 导出稳定

导出安全规则：

- 导出失败不得破坏当前项目内容
- 导出失败不得打断当前编辑状态
- 隐藏对象不得进入导出结果；如素材缺失导致对象无法完整渲染，系统必须明确提示而不是静默导出异常空位
- 当导出尺寸、清晰度或 GIF 时长超出设备能力时，系统必须明确提示用户降级参数

第一阶段不做：

- MP4 导出
- 品牌区
- 社媒导出安全区
- 在线分享链接

### 13. 跨端体验

第一阶段确认：

- Windows 与 Android 都支持完整编辑主链路
- Android 端支持新建项目、打开最近项目、添加和编辑核心对象
- Android 端支持步骤创建、步骤切换、步骤播放、缩放拖拽查看
- Android 端支持本地自动保存、最近项目继续编辑、图片导出
- 平板设备按 Android 端交互策略适配
- 全屏演示模式

平台能力矩阵：

- 两端必备：新建项目、打开最近项目、场地与阵型选择、核心对象编辑、基础线路绘制、步骤编辑、本地保存、PNG 图片导出
- Windows 首发优先：长时间大屏编辑体验、GIF 导出
- Android 首发重点：触控操作、单手可达的核心编辑流程、系统分享图片

后续阶段评估：

- 项目文件
- 备份包
- 快捷键
- 复杂批量操作
- 复杂图层 / 分组管理
- 打印增强
- 球员职责卡
- 战术对比
- Android 端 GIF 导出
- MP4

### 14. 项目组织

第一阶段支持：

- 最近项目列表
- 按最近编辑时间排序
- 基础项目重命名
- 基础项目复制
- 基础项目删除

规则：

- 项目复制作为第一阶段替代版本管理的轻量方案
- 项目删除前必须二次确认，并明确提示删除后不可恢复
- 最近项目记录失效、草稿损坏或引用内容不可恢复时，应用必须允许用户跳过恢复并移除无效条目，不能阻塞进入首页或新建项目

### 15. 编辑效率

第一阶段支持：

- 撤销 / 重做
- 多选
- 对齐 / 吸附
- 锁定 / 隐藏对象

规则：

- 撤销 / 重做作用于当前项目的整体编辑历史
- 切换步骤帧不会清空撤销栈
- 纯查看动作不进入撤销栈，例如步骤切换、缩放、拖动画布、播放暂停
- 会改动内容的动作进入撤销栈，例如移动对象、删除对象、修改文本、调整步骤、调整样式设置

第一阶段不纳入：

- 批量操作
- 快捷键体系
- 图层与分组

### 16. 视觉样式

第一阶段支持：

- 球员展示样式仅支持简洁圆点式、球衣卡片式
- 提供按钮切换球员显示模式
- 第一版不再增加更多球员展示样式
- 主客队一键换肤
- 文本、线路、区域风格预设
- 首版提供 2 套成套主题：简洁战术风 / 复盘讲解风
- 球员展示样式与成套主题相互独立：主题决定球场、文本、线路、区域和整体配色风格，球员仍可单独切换为简洁圆点式或球衣卡片式
- 切换成套主题时默认不强制改变当前球员展示样式，避免误改当前项目的表达方式

### 17. 第一阶段明确不纳入

- 在线账号体系
- 云同步
- 在线分享页 / 分享链接
- 项目文件导入导出
- 本地备份包
- 项目模板 / 球队模板 / 阵容预设 / 球员资料库
- 历史版本
- 模板库
- 快捷键
- 批量操作
- 复杂图层 / 分组
- 打印增强
- 球员职责卡
- 战术对比模式
- Android 端 GIF 导出
- MP4 导出
- 内容创作包装、社媒适配、品牌化导出
- AI 文案辅助

- 海报化视觉包装
- 平台化内容创作模板

## 当前阶段建议

### 第一阶段发布范围（当前建议）

- 先把 `Windows + Android + 离线单机 + 免注册` 的主链路做完整
- 首发重点是战术编辑、步骤演示、本地保存、PNG 导出，以及 Windows 端基础 GIF 导出
- 不要在第一阶段引入文件流转、资产沉淀平台和专业创作包装能力
- 先用“最近项目 + 项目复制”承接基础复用与轻量版本管理

### 讨论原则

- 先判断是否服务于业余足球爱好者的主链路，再决定是否纳入第一阶段
- 遇到会明显扩大实现复杂度的能力，默认先放到后续评估
- 遇到会影响稳定性或数据安全的能力，优先补规则，再决定是否实现
