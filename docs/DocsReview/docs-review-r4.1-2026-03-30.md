## 新观察点

### R4.1-01 editor.md 必备能力新增了"比赛信息与标题区"，但 editor.md §11 验收要点未跟进

问题：

- editor.md §2 必备能力已经补入了"比赛信息与标题区内容编辑"。
- 但 §11 验收要点只写了"核心对象、步骤、保存、导出主链路全部可走通"，没有提及比赛信息的验收标准。

风险：

- 如果质量层验收清单也没有单独覆盖比赛信息（需要和 06-quality 层交叉检查），比赛信息编辑和导出开关可能做了但没有被验收。

影响：低。比赛信息的验收在 R3-2 中已经被补入了质量层 (`acceptance-checklist.md` / `test-matrix.md`)，但 FRD 自身的验收要点未提及，存在微小的追踪断裂。

建议：

- 在 editor.md §11 验收要点中补一条：`比赛信息可编辑，导出时可通过开关控制是否带上`。

### R4.1-02 scope-v1 新增了"Android 图片导出后的系统分享"，但 roadmap 各 Phase 没有显式覆盖

问题：

- scope-v1 §5 纳入清单补了"Android 图片导出后的系统分享"，这很好。
- 但 roadmap.md 的 Phase 3"导出与质量收口"只列了"PNG 导出"和"Windows GIF 导出"，没有提到 Android 系统分享。

风险：

- Phase 3 是导出收口阶段。如果 roadmap 不把 Android 系统分享列入，排期时可能只安排 PNG 文件生成但漏掉分享意图调起（ShareIntent / share sheet），直到 QA 阶段才发现没做。

影响：低。系统分享在实现上通常是 PNG 导出的一个后续步骤，不算独立大工作量，但 roadmap 层有这个缺口可能导致排期遗漏。

建议：

- 在 roadmap.md Phase 3 中补入"Android 图片导出后的系统分享"。
