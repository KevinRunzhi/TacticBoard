# Editor Layout

最后更新：2026-03-30

## 1. 编辑器布局目标

- 让用户把注意力集中在球场和步骤上
- 让高频操作可快速触达
- 让不同设备上的编辑器保持一致心智

## 2. 桌面端布局

固定区域：

- 顶部工具栏
- 左侧工具区
- 中央画布
- 右侧属性区
- 底部步骤栏

布局原则：

- 画布始终是视觉中心
- 属性面板只展示当前上下文相关内容
- 步骤栏在全流程中始终可见
- 导出入口建议固定在顶部工具栏右侧

## 3. 移动端布局

固定区域：

- 顶部轻量工具栏
- 中央画布
- 底部工具栏

抽屉区域：

- 步骤抽屉
- 属性抽屉

布局原则：

- 不让属性面板长期占用画布
- 不让步骤编辑遮挡主要创作区域
- 导出入口建议放在顶部轻量工具栏右侧，保证少步骤可触达

## 4. 返回与退出

- 编辑器必须有明确返回工作台或项目页的入口
- 导出、保存、撤销、播放等高频动作应保持稳定位置
