# User Flows

最后更新：2026-03-30

## 1. 新建项目流

目标：

- 让用户最快从首页进入可编辑状态

路径：

1. 用户进入工作台
2. 点击“新建项目”
3. 进入编辑器
4. 选择场地与阵型
5. 开始编辑
6. 保存或导出

变体路径：

1. 用户进入工作台
2. 点击内置阵型快捷入口
3. 带预设场地与阵型进入编辑器
4. 直接开始编辑
5. 保存或导出

关键规则：

- 不要求登录
- 不要求联网
- 不经过模板中心或球队页
- 通过阵型快捷入口进入时，可跳过“手动选择场地与阵型”步骤

## 2. 最近项目续编流

路径：

1. 用户进入工作台
2. 在最近项目中选择项目
3. 进入编辑器
4. 继续编辑
5. 保存或导出

关键规则：

- 最近项目记录失效时，允许跳过并移除无效条目
- 不允许错误记录卡住首页

## 3. 项目管理流

路径：

1. 用户进入项目页
2. 浏览本地项目列表
3. 执行重命名 / 复制 / 删除
4. 选择项目进入编辑器

关键规则：

- 删除前必须二次确认
- 复制作为轻量版本管理方案
- 不做标签、收藏、复杂筛选
- 不做分享、团队和模板库相关动作

## 4. 导出流

PNG：

1. 用户在编辑器内点击导出
2. 选择导出参数
3. 生成 PNG
4. Android 可继续走系统分享

GIF：

1. 用户在 Windows 编辑器内点击导出 GIF
2. 使用步骤播放结果导出
3. 超出上限时提示减少步骤数量或选择更快速度

## 5. 草稿恢复流

路径：

1. 用户异常退出应用
2. 下次进入工作台或编辑器
3. 系统提示最近一次自动保存草稿
4. 用户可恢复或跳过

关键规则：

- 自动保存失败时必须保留上一份成功草稿
- 恢复失败不能阻塞进入应用

## 6. 素材导入流

头像或参考底图：

1. 用户在编辑器内触发导入
2. 系统文件选择器或平台原生文件选择入口打开
3. Android 端通过最小化授权完成文件访问
4. 应用将素材复制到本地存储
5. 编辑器使用本地副本继续编辑

关键规则：

- 不长期依赖外部文件路径或临时授权引用
- 导入失败时必须提示用户重新选择
- 导入失败时不得留下空引用
- Windows 与 Android 都应遵循“导入后复制到本地”的统一规则
