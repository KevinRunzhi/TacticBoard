# Responsive Rules

最后更新：2026-03-30

## 1. 总原则

- Windows 与 Android 共用同一套页面概念
- 不同设备通过布局变化承接，不通过完全不同的信息架构承接
- 编辑器优先保证画布可用面积

## 2. 工作台 / 项目页 / 设置页

桌面端：

- 允许更宽的内容容器
- 列表与卡片可并列展示

移动端：

- 保持单列主路径
- 一级动作优先放在首屏
- 次级信息放入折叠区或更靠后位置

## 3. 编辑器页

桌面端：

- 三栏或多区布局
- 左侧工具、中间画布、右侧属性、底部步骤栏

平板：

- 保留完整编辑能力
- 重型面板改为抽屉或浮层

手机：

- 画布优先
- 工具与属性放入底栏和抽屉
- 步骤通过底部抽屉进入

## 4. 交互优先级

- 先保证对象选择、移动、步骤切换、保存、导出可用
- 再追求大屏效率增强

## 5. 全屏演示模式

桌面端：

- 允许进入更沉浸的全屏演示状态
- 演示时优先隐藏非必要编辑界面，但保留退出入口

移动端：

- 优先保证画布最大化显示
- 允许弱化系统栏干扰，但必须保留稳定返回路径

规则：

- 全屏演示模式不改变项目数据
- 进入和退出全屏演示不应影响当前步骤和播放状态
