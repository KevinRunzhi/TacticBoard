# Editor FRD

最后更新：2026-03-31
页面：`/editor`、`/editor/:projectId`

## 1. 页面目标

- 承担 V1 核心创作闭环
- 支持场地、阵型、对象、步骤、保存和导出

## 2. 必备能力

- 新建项目
- 打开既有项目
- 场地切换
- 阵型选择与应用
- 球员 / 足球 / 文本 / 区域 / 参考底图编辑
- 线路绘制
- 步骤创建、切换、播放
- 比赛信息与标题区内容编辑
- 本地保存
- PNG 导出
- Windows GIF 导出

### 路由语义

- `/editor`：用于新建空白项目或从快捷入口创建新项目
- `/editor/:projectId`：用于打开既有本地项目
- 访问 `/editor/:projectId` 但 `projectId` 不存在时，进入错误态，不自动回退为新建项目

## 3. 必备区域

- 顶部工具栏
- 工具区
- 画布区
- 属性区或属性抽屉
- 步骤栏或步骤抽屉

### 顶部工具栏

- 返回入口
- 项目名称
- 保存状态提示
- 撤销 / 重做
- 视图相关控制
- 导出入口

### 工具区

- 场地与阵型相关入口
- 对象与线路相关入口

### 属性区

- 只展示当前上下文相关内容
- 无选中对象时展示页面级或步骤级属性
- 页面级属性应包括比赛信息、标题区与导出相关显示开关

### 步骤区

- 展示步骤列表
- 支持新增步骤
- 支持切换当前步骤
- 支持播放 / 暂停

## 4. 核心交互规则

### 场地与阵型

- 新建项目后必须能快速完成场地与阵型初始化
- 切换不兼容制式时必须明确提示
- 制式切换时只提供三个明确选项：
  - 取消切换
  - 保留当前对象内容，仅切换球场制式，当前阵型标记为“自定义”
  - 按新制式重新生成阵型
- 第一阶段不做自动删除多余球员
- 第一阶段不做智能重排
- 第一阶段不做自动映射到新制式

### 对象编辑

- 支持选择、移动、删除核心对象
- 支持对球员、文本、区域、线路进行基础属性调整

### 步骤编辑

- 支持从当前步骤复制到下一步骤继续编辑
- 步骤切换不应清空当前项目的撤销栈
- 支持上一帧残影显示
- 导出时残影默认不导出，需显式开启

### 撤销 / 重做

- 只对内容变更生效
- 纯查看动作不进入撤销栈

## 5. 保存规则

- 自动保存优先覆盖当前工作副本
- 手动保存必须保留当前项目状态
- 自动保存失败时保留上一份成功草稿
- 恢复草稿不应静默覆盖当前项目，需要给出明确恢复入口
- 删除项目不在编辑器内直接完成，统一走项目页或明确确认流程

## 6. 导出规则

- PNG 导出不破坏当前编辑状态
- GIF 仅基于步骤播放结果导出
- 导出失败必须提示，不可静默失败
- 导出时仅导出当前渲染规则允许的内容
- 隐藏对象不应进入导出结果
- 参考底图默认不导出，需显式开启才导出
- 当导出尺寸、清晰度或 GIF 时长超出设备能力时，必须提示用户降级参数，而不是无反馈失败
- 降级提示应引导用户减少步骤数量、降低分辨率或选择更快的播放速度

## 7. 素材导入规则

- 导入球员头像和参考底图时，素材必须复制到应用本地存储
- 不得长期依赖外部文件路径或临时授权引用
- Android 端优先通过系统文件选择器完成最小化授权
- 素材导入失败时必须提示用户重新选择，不得留下空引用

## 8. 平台差异

Windows：

- 优先承载完整多区布局
- 优先承载 GIF 导出

Android：

- 保留完整主链路
- 工具、属性和步骤通过抽屉承接
- 第一阶段不强制承诺 GIF 导出

## 9. 不承接的能力

- 登录引导
- 在线分享链接
- 云同步
- 模板市场
- 团队空间管理

## 10. 非法状态与异常状态

- 项目不存在时，提供返回工作台和项目页的出口
- 素材缺失时，提示重新选择，不保留空引用
- 导出失败时保持当前编辑状态不丢失
- 自动保存失败时给出明确提示，但不阻塞继续编辑

## 11. 验收要点

- 进入编辑器后可以直接开始编辑
- 核心对象、步骤、保存、导出主链路全部可走通
- 比赛信息可编辑，导出时可通过开关控制是否带上
- 错误态不会把用户困在编辑器内
