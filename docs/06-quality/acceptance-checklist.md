# Acceptance Checklist

最后更新：2026-04-23

## 0. 使用规则

- 本清单用于判断当前实现是否真的达到可交付质量，不是“功能看起来差不多”的松散参考。
- 对中型和大型代码改动，执行者必须先自己完成相关验证，再使用本清单判断是否可交付。
- 只跑单个浅层命令、只看 happy path、或只引用旧的验收记录，都不算通过本清单。
- 如果某一项本轮没有验证，应明确写成“未验证”，不能默认视为通过。
- 如果相关自动化测试、桌面验证或手动主链路验证没跑完，本清单不能被用于宣称“已完成验收”。

## 1. 页面结构

- 工作台、项目页、编辑器、设置页、404 页面存在
- 不再暴露登录、注册、分享、团队、模板中心入口
- App Shell 与 Editor Shell 边界清楚
- 旧路由直达时，不再进入旧页面主路径

## 2. 本地项目

- 可以新建项目
- 可以打开最近项目
- 可以在项目页重命名、复制、删除项目
- 删除前有二次确认
- 项目记录失效时可以移除无效条目

## 3. 编辑器

- 可以切换场地
- 可以应用阵型
- 切换到不兼容制式时会弹出三选一提示
- 三个制式切换选项行为正确
- 不会自动删除球员、自动重排或自动映射到新制式
- 可以编辑球员、足球、文本、区域、线路
- 可以切换球员展示样式
- 可以切换成套主题
- 切换成套主题时不会强制改变当前球员展示样式
- 可以创建和切换步骤
- 可以播放步骤
- 步骤切换时上一帧残影显示正确
- 可以撤销与重做内容修改
- 比赛信息可编辑，导出时可通过开关控制是否带上
- 桌面与移动端都能完成完整编辑主链路

## 4. 保存

- 可以手动保存项目
- 自动保存生效
- 异常退出后可提示恢复草稿
- 自动保存失败不破坏已有草稿
- 素材导入后复制到本地存储

## 5. 导出

- PNG 导出成功
- PNG 导出失败时有提示
- PNG 默认不导出参考底图
- PNG 开启参考底图导出后结果正确带上底图
- PNG 默认不导出残影
- PNG 开启残影导出后结果正确带上残影
- Windows GIF 导出成功
- 超出 GIF 限制时有降级提示
- 导出失败不破坏当前编辑状态
- 隐藏对象不进入导出结果
- 比赛信息可按导出开关控制是否带上
- Android 图片导出后可继续走系统分享

## 6. 离线

- 首次打开不要求登录
- 核心页面可在无网络下进入
- 打开最近项目不被更新检查阻塞

## 7. 明确不应出现的问题

- 导出时静默失败
- 自动保存失败后把已有草稿一并破坏
- 素材导入失败后留下空引用
- 旧页面入口重新出现在 V1 导航里

## 8. 中型 / 大型改动附加门槛

- 已运行本轮相关自动化验证，而不是只建议后续运行
- 已运行与改动面匹配的人工主链路验证
- 触及保存、导出、导入、路由、恢复、平台边界时，已覆盖至少一组失败或取消分支
- 已在 `docs/DocsReview/` 记录本轮范围、命令、人工验证、未验证项和剩余风险
- 如果存在阻塞命令或阻塞场景，交付结论里已明确说明，且未把本轮包装成“已充分验收”
