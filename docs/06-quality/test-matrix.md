# Test Matrix

最后更新：2026-04-23

## 使用规则

- 这是最小测试矩阵，不是“只跑其中一两项就可以宣称完成”的快捷菜单。
- 对中型和大型代码改动，必须优先覆盖与本轮改动直接相关的场景，再决定是否还要扩大范围。
- 涉及保存、导出、导入、恢复、路由、平台桥时，不能只跑成功分支。

| 维度 | 场景 | 预期 |
|---|---|---|
| 路由 | 访问 `/` | 进入工作台 |
| 路由 | 访问旧路由 `/login`、`/register`、`/bind-account`、`/share/:id`、`/teams`、`/templates` | 不再进入旧页面主路径，应转为 404 或新路径策略 |
| 路由 | 访问 `/editor/:projectId` 且项目存在 | 正常进入编辑器 |
| 路由 | 访问 `/editor/:projectId` 且项目不存在 | 出现错误态并可返回 |
| 项目 | 新建项目 | 进入可编辑状态 |
| 项目 | 复制项目 | 生成可区分副本 |
| 项目 | 删除项目 | 二次确认后删除 |
| 保存 | 正常自动保存 | 生成最新草稿 |
| 保存 | 自动保存失败 | 保留上一份成功草稿 |
| 保存 | 手动保存后返回工作台再重新打开 | 内容与保存结果一致 |
| 恢复 | 存在草稿 | 启动后可恢复或跳过 |
| 恢复 | 草稿不存在或已失效 | 不崩溃，可继续进入主路径 |
| 编辑 | 切换步骤 | 内容和索引正确 |
| 编辑 | 切换到不兼容制式 | 出现三选一提示，不自动删除、重排或映射对象 |
| 编辑 | 切换球员展示样式 | 球员视觉正确切换 |
| 编辑 | 切换成套主题 | 球场、文本、线路、区域样式变化，球员展示样式保持不变 |
| 编辑 | 切换步骤并查看残影 | 上一帧对象以残影样式显示 |
| 导出 | PNG 导出 | 文件生成成功 |
| 导出 | PNG 导出后文件可打开，结果与当前画布关键内容一致 | 不出现主要对象缺失、错位或静默降级 |
| 导出 | PNG 导出并开启比赛信息 | 输出内容带上比赛信息 |
| 导出 | PNG 默认导出参考底图状态 | 参考底图不出现在导出结果中 |
| 导出 | PNG 导出并开启参考底图 | 参考底图出现在导出结果中 |
| 导出 | PNG 默认导出状态 | 残影不出现在导出结果中 |
| 导出 | PNG 导出并开启残影 | 残影出现在导出结果中 |
| 导出 | 取消导出保存 | 不报错，不破坏当前编辑状态 |
| 导出 | 导出失败 | 有明确提示，不破坏当前编辑状态 |
| 导出 | GIF 超限 | 提示减少步骤或提高速度 |
| 导出 | Android 图片导出后分享 | 可进入系统分享链路 |
| 素材 | 导入头像/底图失败 | 不留下空引用 |
| 素材 | 取消选择头像/底图 | 不报错，当前编辑状态保持可用 |
| 离线 | 无网络启动 | 可进入主页面与编辑器 |
