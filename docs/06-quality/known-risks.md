# Known Risks

最后更新：2026-03-30

## 1. 范围风险

- 编辑器能力容易持续膨胀
- Android 端容易被要求追平桌面重型能力

## 2. 数据风险

- 自动保存失败导致用户误以为已保存
- 素材导入后若不复制到本地，后续会丢失引用

## 3. 导出风险

- GIF 导出比 PNG 更容易出现性能和时长问题
- 导出失败若没有明确提示，会造成“看起来没问题但结果丢失”

## 4. 架构风险

- 若保留旧路由和旧页面入口，前端会继续混入旧产品方向
- 若项目列表、编辑器和持久化边界不清，容易出现状态不同步

## 5. 文档风险

- 若 3 份主文档与 01-07 派生文档不同步，后续实现会出现“按哪个为准”的歧义
- 若 DocsReview 只记录问题不回写主文档，文档会再次漂移
