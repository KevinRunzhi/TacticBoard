## 本轮新增修复亮点

本轮 PRD 除了修复上轮问题外，还主动补强了几条高价值规则：

| 新增内容 | 位置 | 评价 |
|---------|------|------|
| "第一阶段不引入远程模板、远程主题、远程素材拉取等联网依赖" | §5 产品原则 | ✅ 非常好——彻底封闭了 V1 的联网口子 |
| "应用启动、进入编辑器和打开最近项目不得被联网状态检查、远程拉取或更新检查阻塞" | §6.7 规则 | ✅ 非常好——防止了常见的"首次启动卡在检查更新"问题 |
| "Android 端首发不默认申请广泛存储权限，优先使用系统文件选择器完成最小化授权" | §10.2 规则 | ✅ 非常好——符合 Android 最佳实践 |
| "PNG 图片支持透明背景导出" | §6.5 输出增强 | ✅ 比原来的"导出无背景透明图层"措辞更准确 |
| 产品概述补了"Windows 端基础 GIF 导出" | §1 | ✅ 更精确——原来只写"GIF 导出"会让人以为双端都支持 |

## 本轮新发现

| # | 优先级 | 问题描述 | 涉及位置 | 修复状态 |
|---|--------|---------|---------|---------|
| N1 | 🟡 中 | **PRD §6.5 GIF 约束写"超出时提示减少步骤数量或降低停留时长"，但同时写"只提供 慢/标准/快 三档速度"**。"停留时长"这个词暗示用户可以自由配置每帧停留毫秒数，但实际上用户只有三档选择。建议改为"超出时提示减少步骤数量或选择更快的播放速度"，保持与三档速度的口径一致。 | PRD §6.5 | ⬜ 未修复 |
| N2 | 🟡 中 | **PRD §6.7 "当前阶段不纳入第一阶段"这个标题措辞重复**。应为"当前阶段不纳入"或"第一阶段不纳入"。 | PRD §6.7 L322 | ⬜ 未修复 |
| N3 | 🟢 低 | **PRD §16 关键产品模型核心实体列了 `ExportImagePreset`，但 PRD §6.5 的导出增强规则没有提到导出配置是否可保存为 preset**。§6.5 只说"多种导出比例"和"按开关选择是否带上"，没有提及用户是否可以保存自定义导出配置。如果 `ExportImagePreset` 只是内置的几种比例选项，那它的名字可能过于复杂——叫 `ExportConfig` 或者说明它是内置非用户自定义可能更准确。 | PRD §16 vs §6.5 | ⬜ 未修复 |
| N4 | 🟢 低 | **PRD §14 "当前明确不做"（7 项产品方向级）vs Requirements §17 "第一阶段明确不纳入"（18 项实现级）**。两份列表的含义不同但均以"不做"为标题，读者可能混淆。建议在 PRD §13 末尾加一行："更详细的第一阶段不纳入清单，参见 Requirements §17。"或在 PRD §14 加一行："本节为产品方向层面的长期排除项，第一阶段的实现级排除清单见 Requirements §17。" | PRD §14 vs Req §17 | ⬜ 未修复 |
| N5 | 🟢 低 | **Requirements §5 离线保存规则新增了"应用启动…不得被联网状态检查阻塞"，PRD §6.7 也有同样规则**。两处表述一致——确认对齐。但 Requirements §5 的规则条目顺序与 PRD §6.7 不完全相同（Requirements 把联网检查规则放在第 4 条，PRD 放在第 3 条）。不影响正确性，仅做记录。 | Req §5 vs PRD §6.7 | ✅ 已对齐（顺序差异不影响） |
