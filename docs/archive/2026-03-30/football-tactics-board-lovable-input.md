# 足球战术板 Web App - Lovable 页面输入稿

更新日期：2026-03-25
编码要求：UTF-8

## 1. 目标
请基于以下产品定义，为一个 `足球战术板 Web App` 输出页面级 UI 方案。

本次输出重点不是重新定义产品，而是：
- 按既定信息架构做页面设计
- 保持页面边界清晰
- 让编辑器足够专业、沉浸
- 让资产页足够清楚、易管理
- 让分享页足够轻、易传播

不要自行发明新的产品模块，不要改动页面职责，不要把多个页面合并成一个“大而全后台”。

## 2. 产品定位
- 产品类型：Web App
- 核心方向：足球战术板编辑器
- 目标用户：
  - 足球爱好者
  - 足球内容创作者 / 自媒体
  - 业余球队 / 队长 / 教练
- 核心价值：
  - 专业表达
  - 好看出图
  - 方便分享
  - 长期资产沉淀

## 3. 信息架构总原则
- 首页 / 落地页 与 应用内页面分开
- 编辑器页 与 分享页使用沉浸式独立页面
- 编辑器只负责创作，不承担重资产管理
- 阵容预设、球员资料库都挂在球队页下面，不做顶级导航
- 模板中心是一级页面
- 个人空间 / 团队空间通过全局空间切换器切换

## 4. 前端结构建议
请按以下结构理解页面壳层：

- `Marketing Shell`
  - 首页 / 落地页
  - 登录 / 注册

- `App Shell`
  - 工作台
  - 项目页
  - 模板中心
  - 球队页
  - 设置页

- `Immersive Shell`
  - 编辑器页
  - 分享页

补充要求：
- 编辑器页与分享页不要复用普通后台壳层
- 编辑器页要有明确的“返回工作台 / 返回项目页 / 返回上一资产列表”出口
- 空间上下文建议体现在页面结构中，不要只靠临时状态

## 5. 核心页面清单
你需要为以下 8 个页面输出 UI：

1. 首页 / 落地页
2. 工作台
3. 项目页
4. 模板中心
5. 球队页
6. 编辑器页
7. 分享页
8. 设置页

## 6. 页面职责
### 6.1 首页 / 落地页
职责：
- 产品介绍
- 登录 / 注册
- 游客直接开始
- 进入分享内容

### 6.2 工作台
职责：
- 登录后的默认入口
- 最近编辑
- 继续编辑
- 快速新建
- 前往项目、模板、球队等核心页面

### 6.3 项目页
职责：
- 当前空间下项目资产的主管理页
- 搜索、筛选、置顶、标签
- 新建项目
- 进入编辑器
- 复制、副本、分享、删除

### 6.4 模板中心
职责：
- 官方模板库
- 私有模板库
- 项目模板
- 阵型模板
- 模板预览与复用

### 6.5 球队页
职责：
- 球队模板管理
- 阵容预设管理
- 球员资料库管理
- 主客场视觉配置

### 6.6 编辑器页
职责：
- 创作战术板
- 摆阵型
- 绘制线路和区域
- 管步骤
- 管图层
- 导出和分享

限制：
- 不承担模板、球队、球员资料的重管理
- 只允许轻量套用模板 / 阵型 / 球队资产

### 6.7 分享页
职责：
- 只读查看
- 播放步骤
- 缩放查看
- 复制为副本

限制：
- 不提供完整编辑能力
- 不与编辑器混用

### 6.8 设置页
职责：
- 个人资料
- 登录方式
- 安全设置
- 默认空间与创建偏好
- 账号退出 / 注销

## 7. 页面跳转关系
请按以下关系组织页面：

- 首页 -> 登录 / 注册 -> 工作台
- 首页 -> 直接开始 -> 游客态编辑器
- 首页 / 外部链接 -> 分享页
- 工作台 -> 新建项目 -> 编辑器页
- 工作台 -> 最近编辑 / 继续编辑 -> 编辑器页
- 工作台 -> 项目页 / 模板中心 / 球队页 / 设置页
- 项目页 -> 打开项目 -> 编辑器页
- 项目页 -> 新建项目 -> 编辑器页
- 模板中心 -> 套用项目模板或阵型模板 -> 编辑器页中的新项目
- 球队页 -> 基于球队模板、默认阵容预设或手动选择阵容发起新建 -> 编辑器页
- 编辑器页 -> 生成分享链接 -> 分享页
- 分享页 -> 复制为副本 -> 登录或绑定账号 -> 编辑器页中的新副本项目

## 8. 核心用户流程
### 8.1 游客试用流程
- 首页
- 直接开始
- 游客态编辑器
- 触发账号能力
- 登录绑定
- 工作台或项目页

### 8.2 模板驱动创作流程
- 工作台或模板中心
- 选择官方项目模板 / 私有项目模板 / 阵型模板
- 系统按覆盖顺序生成新项目
- 编辑器页
- 保存 / 导出 / 分享

### 8.3 球队战术创作流程
- 切换到目标空间
- 球队页
- 选择球队模板与阵容预设
- 编辑器页
- 保存项目
- 分享或导出

### 8.4 分享后二次创作流程
- 分享页
- 复制为副本
- 登录或绑定账号
- 编辑器页中的新副本项目
- 继续编辑并另行保存

## 10. 关键功能边界
### 10.1 游客态编辑器
支持：
- 基础创作
- 本地草稿保存
- 步骤编辑
- 图片导出

不支持：
- 云端保存
- 分享链接生成
- 个人 / 团队空间资产写入
- 球队模板 / 阵容预设套用

触发这些能力时，需要弹出登录 / 注册引导。

### 10.2 手机端边界
手机端只做轻编辑，不做完整编辑。

支持：
- 切换步骤
- 播放步骤
- 缩放拖拽查看
- 微调球员 / 足球位置
- 修改步骤标题与文字说明
- 图片导出

不支持：
- 新建复杂线路
- 新建区域
- 图层和分组管理
- 模板管理
- 资产管理
- GIF / MP4 导出

### 10.3 编辑器边界
编辑器不承担：
- 球队模板重管理
- 阵容预设重管理
- 球员资料重管理
- 模板库重管理

编辑器只允许：
- 轻量套用模板
- 轻量套用阵型
- 轻量套用球队 / 阵容资产

## 11. 视觉方向要求
不要做成普通后台系统，也不要做成泛白板工具。

请遵守这些视觉要求：
- 页面要有明确的足球产品气质
- 编辑器要专业、沉浸、清爽
- 分享页要轻、干净、适合手机端传播
- 资产页要像成熟 SaaS，不要像杂乱 CMS

额外限制：
- 不要用紫色作为主色倾向
- 不要做默认白底 + 紫色按钮那种常见 AI 风格
- 不要只用系统字体
- 不要做平庸的后台模板观感

建议方向：
- 用清晰的层次、强对比的功能区、少量但明确的足球草坪 / 场地语义元素
- 可以用 CSS 变量组织主题
- 动效以“有意义的切换和进入”为主，不要堆无意义微动效

## 12. 组件抽象建议

- 副标题
- 三点操作菜单

可派生变体：
- `ProjectCard`
- `TemplateCard`
- `TeamCard`
